# 后端自动启动配置指南

## 方法 1：使用 Windows 任务计划程序（推荐）

### 步骤：

1. **打开任务计划程序**
   - 按 `Win + R`，输入 `taskschd.msc`，按回车

2. **创建基本任务**
   - 右侧面板点击"创建基本任务"
   - 名称输入：`启动拾物集后端`
   - 描述：`在用户登录时自动启动拾物集后端服务`

3. **触发器设置**
   - 选择"当用户登录时"
   - 点击"下一步"

4. **操作设置**
   - 选择"启动程序"
   - 程序或脚本：`C:\Windows\System32\cscript.exe`
   - 添加参数：`start-backend-auto.vbs`
   - 起始位置：`e:\大学\item-manager-page-three`
   - 点击"下一步"

5. **完成**
   - 勾选"开启此任务"
   - 点击"完成"

---

## 方法 2：快速启动方式

1. **创建快捷方式**
   - 右键点击 `start-backend-auto.vbs`
   - 选择"发送到" → "桌面（快捷方式）"

2. **添加到启动文件夹**
   - 按 `Win + R`，输入 `shell:startup`，回车
   - 将 `start-backend-auto.vbs` 复制到这个文件夹

---

## 方法 3：使用 PM2 进程管理器（最稳定）

1. **全局安装 PM2**
   ```powershell
   npm install -g pm2
   ```

2. **启动后端服务**
   ```powershell
   cd e:\大学\item-manager-page-three\backend
   pm2 start server.js --name "shiwuji-backend"
   ```

3. **设置开机启动**
   ```powershell
   pm2 startup windows
   pm2 save
   ```

4. **验证**
   ```powershell
   pm2 list
   ```

---

## 检验是否正常工作

- 重启电脑后，检查后端是否已启动
- 在浏览器打开 http://localhost:3000
- 或在命令行运行：`netstat -ano | findstr :3000` 检查端口占用

## 故障排除

如果后端没有自动启动：

1. **检查 npm 环境**
   ```powershell
   npm --version
   ```

2. **手动运行脚本测试**
   ```powershell
   cd e:\大学\item-manager-page-three
   powershell -ExecutionPolicy Bypass -File start-backend-auto.ps1
   ```

3. **查看任务计划程序日志**
   - 任务计划程序 → 任务计划程序库 → 右键任务 → 属性 → 历史记录

4. **收集错误信息**
   - 修改 `start-backend-auto.ps1` 末尾添加：
     ```powershell
     "启动时间：$(Get-Date)" | Add-Content "C:\temp\backend-start.log"
     ```
